# OpenShare

<p align="center">
  <img src="assets/banner.svg" alt="OpenShare Banner" width="100%">
</p>

OpenShare 是一个面向校园网、实验室、社团、研究组和中小型组织内网环境的自托管资料共享平台。它的目标不是做一个面向公网的大型内容社区，而是把“资料上传、审核发布、检索浏览、下载分发、运营治理”这条链路在一个轻量系统里做完整。

项目采用前后端分离开发、单二进制发布的思路：前端使用 Vue 3 构建管理端与公开站点，后端使用 Go + Gin 提供 API、静态资源托管、权限控制、审核流转和本地文件存储管理。默认数据层使用 SQLite，适合快速部署和低维护成本场景。

## 适用场景

- 校园课程资料、往年题、实验报告共享
- 实验室内部文档、数据样例、工具包沉淀
- 研究组服务器资料归档与统一检索
- 社团、组织内部学习材料共享
- 局域网或内网文件分发平台

如果你的目标是快速搭一个可控、可审计、低依赖的资料库，而不是一个复杂的开放式内容平台，OpenShare 更贴合这个方向。

## 项目特点

- 公开侧默认免注册，访问门槛低
- 支持访客上传资料，并通过审核流程决定是否发布
- 支持单文件、文件夹和批量资源下载
- 支持回执码机制，查询上传记录和反馈处理结果
- 支持公告、反馈、资源编辑、资源下架、资源删除
- 支持管理员账号、权限、会话和操作日志
- 支持从服务器本地目录导入已有资料
- 默认使用 SQLite + 本地文件系统，部署简单

## 功能总览

### 1. 公开站点能力

OpenShare 的公开站点面向普通访客，不要求用户注册账号即可使用。

#### 首页与目录浏览

- 展示最新公告，支持置顶公告
- 展示热门下载和最新资料
- 展示根目录与子目录结构
- 支持卡片视图和表格视图切换
- 支持按名称、下载量、格式排序
- 支持目录面包屑导航
- 支持目录详情、目录描述和目录统计信息展示

#### 搜索与检索

- 提供统一搜索入口
- 搜索对象覆盖文件和目录
- 支持按关键词检索公开资源
- 支持目录范围限定搜索
- 支持模糊匹配与结果窗口控制
- 搜索能力由后端索引维护，可手动重建索引

#### 文件详情与下载

- 查看文件标题、原始文件名、类型、大小、更新时间、下载次数
- 支持单文件下载
- 支持整目录下载
- 支持批量下载多个资源
- 下载行为会累计下载次数，并记录下载事件用于后台统计

#### 访客上传

- 访客可以直接上传单个文件或带目录结构的文件集合
- 上传前生成或同步回执码，用于后续查询状态
- 上传内容先写入暂存区，再进入审核流程
- 可根据系统策略决定是“直接发布”还是“必须审核”
- 支持文件大小限制
- 后端对上传接口配置了限流能力

#### 回执码查询

公开侧提供独立的回执查询页，用户无需账号即可查看：

- 自己的上传记录
- 当前上传状态：待审核、已通过、已驳回
- 驳回原因
- 自己提交的反馈记录
- 反馈状态：待处理、已处理、已驳回

回执码会在会话中自动维护，也可以手动清除或重新查询。

#### 反馈与举报

访客可以针对文件发起反馈：

- 提交问题描述
- 获取反馈回执码
- 后续通过回执码查询处理结果

后台审核人员可以决定：

- 通过反馈并记录处理说明
- 驳回反馈并记录理由

### 2. 管理后台能力

OpenShare 提供一个独立的管理后台，使用 `Session + Cookie` 进行认证。

#### 控制台

后台首页展示关键运营指标：

- 总访问 IP 数
- 总资料数
- 总下载数
- 近 7 天访问 IP 数
- 近 7 天新增资料数
- 近 7 天下载数

这些指标来自访问记录、文件记录和下载事件。

#### 上传审核

管理员可在审核页查看待审核上传：

- 查看标题、描述、文件名、大小、MIME 类型
- 查看上传时间与回执码
- 查看目录结构快照
- 通过审核，将文件从暂存区转入正式资料区
- 驳回审核，并填写驳回原因

#### 反馈审核

管理员可查看待处理反馈：

- 查看反馈目标是文件还是目录
- 查看反馈说明、提交时间、提交来源 IP
- 批准反馈并记录处理意见
- 驳回反馈并记录理由

#### 资源管理

后台和公开详情页共同支持部分资源治理操作，取决于权限配置：

- 查看资源文件列表
- 编辑文件标题、扩展名、描述
- 编辑目录描述
- 下架文件
- 删除文件
- 删除目录

删除和部分高风险操作需要再次验证管理员密码。

#### 公告管理

管理员可维护首页公告：

- 创建公告
- 编辑公告
- 删除公告
- 设置状态为草稿、已发布、隐藏
- 超级管理员可设置置顶
- 公告内容支持 Markdown 渲染

#### 管理员管理

具备管理员管理权限的账号可以：

- 查看管理员列表
- 创建新管理员
- 分配权限
- 启用或停用管理员
- 重置其他管理员密码
- 删除管理员

当前实现中的核心权限维度包括：

- `submission_moderation`：上传审核
- `resource_moderation`：反馈处理、资源编辑、删除、下架
- `announcements`：公告管理
- `manage_admins`：管理员管理
- `manage_system`：系统设置、本地目录导入、索引维护

#### 账号与会话

后台账号相关能力包括：

- 管理员登录和退出登录
- 查询当前登录账号信息
- 修改个人资料
- 修改个人密码
- 基于持久化 Session 的登录状态续期

#### 系统设置

超级管理员可维护系统策略：

- 是否允许访客直发
- 是否允许访客编辑标题
- 是否允许访客编辑描述
- 是否允许访客删除资源
- 上传大小上限
- 搜索策略参数

#### 本地目录导入

系统支持将服务器上已有目录纳入 OpenShare 管理：

- 浏览服务器目录
- 选择目标目录进行导入
- 建立目录与文件元数据
- 纳入统一检索与下载体系
- 查看已托管目录
- 删除已托管目录映射

这适合已有历史资料目录的场景，不需要重新上传一遍文件。

#### 操作日志

系统会记录关键后台操作，用于审计：

- 创建、修改、删除管理员
- 重置管理员密码
- 修改个人资料和密码
- 通过/驳回上传审核
- 处理/驳回反馈
- 更新资源、下架资源、删除资源
- 导入本地目录
- 更新系统设置
- 创建、修改、删除公告

每条日志会记录：

- 操作人
- 操作类型
- 对象类型
- 对象标识
- 详情
- 来源 IP
- 时间

## 角色与权限模型

### 访客

- 浏览目录
- 搜索资源
- 查看文件详情
- 下载文件或目录
- 提交上传
- 查询回执
- 提交反馈

### 管理员

管理员能力取决于权限组合，适用于日常审核与运营。

### 超级管理员

超级管理员拥有更高权限，负责：

- 管理系统设置
- 管理管理员账号
- 删除托管目录
- 执行更高风险治理操作

## 关键业务流程

### 上传到发布

1. 访客上传文件或目录
2. 系统生成/绑定回执码
3. 文件写入 `staging` 暂存区
4. 后台审核通过或驳回
5. 通过后写入正式资料区并建立元数据
6. 驳回后保留审核结果，用户可用回执码查询

### 反馈到处理

1. 访客在文件详情页提交反馈
2. 系统记录反馈、目标对象、描述和来源信息
3. 管理员在审核页处理反馈
4. 用户通过回执码查询最终状态

### 本地目录导入

1. 超级管理员浏览服务器目录
2. 选择待导入目录
3. 系统扫描目录树并建立 `folders/files` 元数据
4. 导入资源进入公开检索与下载体系
5. 后续可作为托管目录继续管理

## 存储与数据设计

OpenShare 采用“本地文件系统保存内容，SQLite 保存元数据”的方案。

### 本地文件系统

- 正式资料存放在存储根目录下
- 上传暂存文件放在 `staging/`
- 清理或回收文件可进入 `trash/`

### SQLite 元数据

系统主要维护以下实体：

- `admins`：管理员账号
- `admin_sessions`：后台会话
- `folders`：目录树
- `files`：文件元数据
- `submissions`：上传记录
- `reports`：反馈记录
- `announcements`：公告
- `operation_logs`：操作日志
- `site_visit_events`：访问记录
- `download_events`：下载记录
- `system_settings`：系统策略

### 资源状态

资源与审核对象的状态模型清晰区分：

- 上传记录：`pending / approved / rejected`
- 资源：`active / offline / deleted`
- 反馈：`pending / approved / rejected`
- 公告：`draft / published / hidden`

## 搜索设计

当前搜索服务由后端统一维护索引，支持：

- 文件与目录统一检索
- 目录范围检索
- 模糊匹配
- 手动重建索引

搜索策略由系统设置控制，适合中小规模资料库，不追求外部搜索引擎级别复杂能力，而是以低依赖、可用性优先。

## 配置说明

后端启动时会依次加载：

- `backend/configs/config.default.json`
- `backend/configs/config.local.json`

本地配置会覆盖默认配置。默认关键项包括：

- 服务监听地址与端口
- SQLite 数据库路径
- 存储根目录、暂存目录、回收目录
- 上传大小限制
- Session 名称、密钥、Cookie 策略
- 上传和搜索接口限流策略

示例默认值中：

- 服务端口：`8080`
- 数据库路径：`/data/openshare/openshare.db`
- 存储根目录：`/data/openshare`

## 本地开发

### 前置依赖

- Go
- Node.js / npm
- curl
- lsof

### 一键启动开发测试环境

项目提供脚本：

```bash
bash scripts/dev-test.sh
```

这个脚本会：

- 重置 `.localdata/` 下的本地数据
- 自动生成 `backend/configs/config.local.json`
- 检查并安装前端依赖
- 启动后端服务
- 启动前端开发服务器
- 输出超级管理员初始账号信息

启动后默认访问地址：

- 公开站点：`http://localhost:5173/`
- 管理后台：`http://localhost:5173/admin`

注意：这个脚本偏向删档测试，每次运行都会清空本地数据库和本地存储目录。

### 手动启动

后端：

```bash
cd backend
go run ./cmd/server
```

前端：

```bash
cd frontend
npm install
npm run dev
```

## 构建与发布

### 本机构建 Linux 版本

```bash
bash scripts/build-linux.sh
```

脚本会执行以下步骤：

- 检查并安装前端依赖
- 构建前端产物
- 同步前端 `dist` 到 `backend/web/dist`
- 编译 Linux AMD64 后端二进制

构建输出默认位于：

```text
release/linux-amd64/openshare
```

### 通过 Docker 构建 Linux 版本

如果本机环境不方便直接编译 Linux 二进制，可以使用：

```bash
bash scripts/build-linux-docker.sh
```

## 默认运行行为

- 启动时自动确保数据库 schema 已初始化
- 启动时自动检查存储目录结构
- 启动时自动创建默认超级管理员
- 启动时自动初始化搜索索引
- 后端同时托管 API 与嵌入式前端资源

## 当前实现边界

OpenShare 当前更偏向“内网资料共享系统”，而不是复杂的云盘或内容社区。它的边界大致是：

- 没有普通用户注册体系
- 不做复杂社交关系和互动系统
- 不依赖对象存储、消息队列、外部搜索服务
- 不追求超大规模分布式部署

这也是它能保持轻量和易部署的原因。

## 总结

OpenShare 的核心价值不在于“功能特别多”，而在于把一个资料共享平台真正需要的关键环节串起来：

- 访客可以低门槛上传和下载
- 管理员可以审核、治理和追踪操作
- 超级管理员可以配置系统和接管导入
- 文件内容和元数据分离，便于本地部署和长期维护

如果你需要的是一个适合校园网、实验室、团队内网的自托管资料共享平台，OpenShare 的当前能力已经覆盖了从上传到治理的主链路。
